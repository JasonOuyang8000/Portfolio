CryptoPlay Version 1

Notable Technologies

FrontEnd:

  • VANILLA JS
  • MODALS
  • CHART JS
  • PURE CSS

Backend:

  • BCRYPT
  • EXPRESS.JS
  • JWT TOKEN
  • SEQUELIZE
  • COIN RANK API

KEY FEATURES:

  1. User can search a crypto which then shows at most 8 of the results from the COIN RANK API.
  2. When user clicks on a certain crypto, whether it's in the chart or from the search result, it should display a year's graph of the crypto.
  3. Buying and selling crypto is made as simple as possible. When buying a crypto, user will be prompt to enter dollar amount. For selling crypto, user will be prompt to enter crpyto ammount. There is a buy all or sell all for one crypto to make it even simpler.
  4. Each transaction will first ask user to confirm the purchase/sell with the current price.
  5. There is a dashboard page which shows the User's owned crypto and user's estimated portfolio value. The portfolio value is calculated is getting the current price for each crypto the user owns by the ammount they have.

Crypto Play is a web application which allows users to buy/sell crypto for a made up balance. Crypto prices are calculated via prices from the COIN RANK API.

Socialfy

Notable Technologies

FrontEnd:

  • REACT JS
  • BOOTSTRAP 5.0
  • SOCKET IO 4.1
  • REACT VIS
  • IMAGE UPLOAD
  • LOCAL STORAGE

Backend:

  • BCRYPT
  • EXPRESS.JS
  • JWT TOKEN
  • SEQUELIZE
  • AWS SDK
  • MULTER S3
  • SIMPLE ENCRYPTOR

KEY FEATURES:

  1. A User can write a post that can be either text, image, or text and image. Image is through file upload and handled using AWS S3 in the backend.
  2. Logging in is done through using a secret key from a succesful signup.This provides more security for the user. The key is created using encryption from the backend.
  3. A User can like their own post or other users' post. Each post can have a number of likes. Each user can only give one like. Liking again will remove the like from the post.
  4. A User can also comment on a post. Comments can be shown when user clicks on the comment icon the the post. The comment icon will also show the amount of coments for the post.
  5. There is a friend list built in for each user. For a user to be friends with another user, they must first go to their profile and hit send friend request.The other user would then need to accept the friend request. A User can also cancel a friend request which will make the other user not see the friend request sent.
  6. Once a friend is added, a user can then message that friend from their friend list. It is done through self to self assocations in the backend.

Socialfy is a web application which allows you to write and comment posts with other users. Users can also add other users as friends and then message them privately.

Team Chat

Notable Technologies

FrontEnd:

  • REACT JS
  • BOOTSTRAP 5.0
  • SOCKET IO 4.1
  • REACT VIS
  • MOMENT JS
  • LOCAL STORAGE

Backend:

  • BCRYPT
  • FLASK 2.0
  • JWT TOKEN
  • SQL ALCHEMY
  • FLASK SOCKET IO 5.3

KEY FEATURES:

  1. Channel Alerts in Channel if user has not seen a message sent by another user.
  2. Newly created workspaces by another user will prompt user to reload page if user is in workpaces page.
  3. Workspaces can be protected with a password.This can provide privacy for users who want to chat amongst their friends.
  4. Every Workspace has an image and name to distinguish itself from other workspaces.When creating a workspace, the user can take advantage of the Image Search Library which uses the PEXELS Image API. Every successful search will spawn the relevant images with client side pagination.
  5. A Dashboard Page showing Users the amount of messages sent for the current week. The user can also change their password or increase their workspace limit here.
  6. When a User is logged in, a token is added to local storage to save the user's login session. If a user enters a workspace, a workspace token is created to save the user's current workspace session.

Team Chat is Web Application allowing users to connect and chat with one another.Users can do this by creating an account and then accessing or creating their own workspace. The workspace is where the User can connect and chat with other users.
